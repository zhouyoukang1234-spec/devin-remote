## 本次修复的关键 Bug

### dao-vsix 全功能面板空白（三个串联 bug）
1. **渲染层**：`getDaoCloudMiddlePanelHtml` 内重复粘连垃圾片段 → `SyntaxError: Unexpected token '&'` → 整段脚本不解析。
2. **渲染层**：webview 的 `esc()` 调用服务端函数 `mpEsc` → `ReferenceError: mpEsc is not defined` → 内容区空白。
3. **数据层**：`devinListSessions` 用 `ws.devinApiKey || auth1` 选 token，误用 189 字 Codeium session-token 打 `app.devin.ai/v2sessions`（返 401）。修法：
   ```ts
   const useV1Api = (ws.devinApiKey || '').startsWith('cog_');
   const apiKey = useV1Api ? ws.devinApiKey : auth1;
   ```

### Devin Desktop 3.1.7 Cognition 登录闸门
- 新版引入 Auth0 原生登录门，需 `app.devin.ai/auth/windsurf/show-auth-code` 的 `ott$` token（浏览器登录后获取，官方"Having trouble?"手动兜底流程）。

### Cloudflare 1010（内网穿透）
- relay 请求必须带浏览器 `User-Agent`，否则 Cloudflare 返回 403 error 1010。

## Token 类型速查
| 前缀 | 用途 |
|------|------|
| `devin-session-token$...` | rt-flow 注入；仅 Codeium/Windsurf API 可，打 app.devin.ai 返 401/403 |
| `ott$...` | app.devin.ai 一次性授权码，过 3.1.7 登录闸门 |
| `cog_...` | Devin 5 v1 API Key |
| `auth1` | Devin Cloud 会话令牌，app.devin.ai 端点用它 |
