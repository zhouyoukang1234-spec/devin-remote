## 二、关键增量经验（在 runbook 之上）

1. **Devin API 真实鉴权链（重大突破）**: 网页 cookie ≠ API 鉴权。API 走 Bearer auth1:
   `POST https://windsurf.com/_devin-auth/password/login {email,password}` → `token: auth1_...`
   端点: `/api/org-<bare_id>/v2sessions` (≠v2/sessions)、`/secrets`、`/playbooks`、`/learning/all`
2. **dao-bridge 是更稳的 141 通道**: workers.dev DurableObject relay
   `https://dao-relay-do.zhouyoukang.workers.dev/relay/141` Bearer `dao141-9c2e7a1f4b6d8035`
   ⚠ 必须带 User-Agent 头，否则 CF 403。trycloudflare 快速隧道已废弃（卡死）。
3. **8937 本地代理是明文 HTTP**: Node 里对 127.0.0.1 要用 `http` 模块；windsurf.com 经代理 502 → 直连优先+代理回退。
4. **write141 中文路径坑**: exec 的 stdout 回传中文会 GBK 损毁，把损毁名再拼路径写入会建出乱码目录。
   **解**: 路径解析在 141 侧 PS 内完成（写临时文件再 PS Move-Item），或全 base64 round-trip。
5. **VSIX 二进制传 141**: base64 分块(400KB) write141 → 141 侧 `[Convert]::FromBase64String` 重组。
