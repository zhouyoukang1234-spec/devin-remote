# 道法自然 · 6 插件小白上手指南（配套操作视频）

> 「上士聞道，堇而行之。」本指南配合 6 个操作视频，带你从零跑通 devin-remote 各单体能力。
> 每一步都在真实虚拟机上实测通过，照着做即可复现。
>
> 📌 本指南是**单体插件逐个走读**（便于理解每个能力）。仓库当前是 **4 核心 + 4 辅助 + 3 板块** 架构，**最终交付是把四引擎归一的单插件 dao-one**（日常只装它一个即可，见仓库 README）。下面的 ①–⑥ 是其源构件的单独形态。

## 0. 冷启动（一次性，5 分钟）

仓库已把整套安装脚本化。在一台干净的 Windows 机器上：

```powershell
git clone https://github.com/zhouyoukang1234-spec/devin-remote.git $env:USERPROFILE\repos\devin-remote
powershell -ExecutionPolicy Bypass -File $env:USERPROFILE\repos\devin-remote\cloud\coldstart\coldstart.ps1
```

脚本会自动：① 下载并静默安装 Devin Desktop ② 从源码构建 **dao-one** 归一插件（VSIX 是 gitignored 的，需现场编译）③ 安装后让 `dao.dao-one` 成为唯一属主。
完成后启动 IDE：`%LOCALAPPDATA%\Programs\Devin\Devin.exe`。

> 下文 ①–⑥ 演示的是**各单体插件**的独立形态；日常使用以 dao-one 单插件为主（同一面板内已内联这些能力）。

> 账号池不在仓库里，由你自己提供（邮箱:密码）。首个账号由 rt-flow 登录，拿到 auth1，绕过浏览器 OAuth。

## ① dao-vsix · 全功能面板（视频 1）

1. 点活动栏的 Devin Cloud 图标，打开面板。
2. 面板顶部显示已登录账号 / 组织 / 配额。
3. 点「打开全功能面板」——会加载真实会话、Playbook 列表。
4. 终端验证本地 API：`curl http://localhost:9920/api/health` 返回 `status: ok`。

> **关键坑（已实测）**：全功能面板（Sessions/Knowledge/Playbook/Secret）需要 **auth1**，而 IDE 自动注入的只是 session-token（对 v2 API 会 403）。
> 解法：在面板里用「邮箱+密码」登录拿到 auth1，之后 106 会话 / 175 Playbook / 2 Secret 全部正常。

## ② dao-bridge · 内网穿透（视频 2）

> 现版本 **relay-first**：默认走 dao-relay 零账号 Worker 中继（URL 天然稳定、无需任何 Cloudflare 账号），连不上才自动回退 cloudflared。视频里的 `trycloudflare` 地址即回退档；如需强制只用 cloudflared 可开 `daoBridge.disableRelay`。

1. 打开 dao-bridge 面板，点「打通公网」。
2. 面板显示公网地址并变为「已打通·公网在线」：默认是 `https://dao-relay-do.<sub>.workers.dev/relay/<session>`（中继档），回退时是 `https://....trycloudflare.com`（cloudflared 档）。
3. 验证：`curl https://<你的地址>/api/health` 返回 `{status:ok, service:dao-bridge}`。
4. 纯 Node / 手机端：无 VS Code 的环境用 `addons/dao-bridge`（`start.ps1`）或 `addons/dao-bridge-android`（Termux）跑同一中继。

## ③ devin-git-auth · 多账号归一（视频 3）

1. 活动栏「⋯ 更多」→「Git Auth」打开面板。
2. **GitHub PAT · 归一连接目标**：粘贴你的 GitHub PAT（字段自动掩码，安全），点「保存」。
3. **添加账号 (Devin)**：填邮箱 + 密码，点「登录单个」——账号进入下方列表（显示组织、状态「无Git/未连」）。
4. 点「**全部连接到同一 GitHub**」——状态变为「● Git:1 / 已连1」，即该 Devin 账号已绑定到同一个 GitHub。
5. 多个账号重复 3-4，即可把整池 Devin 账号都连到同一 GitHub。

## ④ dao-proxy-pro · 本源观照（视频 4）

1. 活动栏「道Agent Pro」打开「本源观照」面板。
2. 面板显示已注入的**底层系统提示词**（如「帛书老子+道藏阴符经」，7846 字）。
3. 顶部下拉可在 3 套预设间切换：帛书老子+道藏 / 帛书《老子》/ 道藏《阴符经》。
4. 多模型路由后端：`curl http://localhost:8937/origin/ea/overview` 返回 `ok=true`，覆盖 8 个家族（Claude/DeepSeek/GPT/Gemini/Grok/Kimi/Qwen/SWE）。

## ⑤ rt-flow · 多账号备份（视频 5）

1. 活动栏「⋯ 更多」→「RT Flow」打开面板。
2. 顶部汇总额度（如 D300 W252 $13），下方「账号列表」逐个显示实时额度（D100 / W99…）。
3. 点「**全部备份**」——把所有账号的对话增量备份成本地 ZIP。
4. 备份位置：`~/.wam/devin_cloud_backups/<邮箱>/`（每个对话一个 zip）。

## ⑥ dao-export · 对话全量导出（视频 6）

纯零依赖 Python 脚本，**仅凭邮箱+密码**导出一切。为避免密码上屏，把账号写进文件用 `--accounts`：

```powershell
# accounts.md 每行一个 email:password
cd cloud\export-accounts
python dao_export_all.py --accounts C:\path\to\accounts.md --output C:\path\to\out
# 单账号：python dao_export_all.py --email you@x.com --password ***
# 限量预览：加 --max-sessions 3
```

导出内容：账号级 secrets / playbooks / knowledge / git_connections / org_settings + 全部会话（每个会话含 `session_info.json`、`worklog`、`changes`、`cloud_files`、`file_manifest.json`）。实测一个账号 106 会话 ≈ 50MB。

## 排错速查

| 现象 | 原因 / 解法 |
|---|---|
| 面板「加载中」卡住 | 旧坑：模板字符串里正则反斜杠折叠 / webview 屏蔽 confirm。现版本已修复。 |
| 全功能面板接口 `{ok:false}` | 用的是 session-token，不是 auth1。请在面板用邮箱+密码登录。 |
| 进程重启后后端没了 | 磁盘文件还在，重新启动 `Devin.exe`，插件后端会自动拉起。 |
| 输入框丢字符（@、冒号） | 在 webview/终端用剪贴板粘贴（Ctrl+V），不要逐字键入。 |
