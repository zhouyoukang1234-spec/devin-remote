## 1. dao-proxy-pro v9.9.261 — 外接第三方模型路由

### 架构 (已落地)
```
extension.js (203KB, shell/webview)   ──spawn hook──▶ source.js (反代核心 :8937)
   · 三面板 webview UI                                   · /origin/* 路由
   · _EA_PRESETS (14家 cc-switch 预设)                   · MODEL_UNLOCK 拦截注入
   · 连线面板 eaDrawWires / $eaRight                      · 调 vendor/外接api/runtime.js
                                                          ▼
                          vendor/外接api/core/dao_router.js (150KB 路由核心)
                            · shouldRoute / route / extractModelUid
                            · hotAddProvider / hotAddRoute / unlockModel
                            · adapters.js (provider 适配) · cascade_wire.js (protobuf 编解码)
                            · 配置: ~/.codeium/dao-byok/配置.json
```

### 需求对照
| 需求 | 状态 | 证据 / 说明 |
|---|---|---|
| #1 Cascade 显示全部模型，免费/Pro 账号一致 | **已具备** | `source.js` `MODEL_UNLOCK`: 拦截 `GetUserSettings`/`GetCascadeModelConfigs` 响应 → 注入全量 109 模型目录 (`_full_model_catalog.json`)。注入仅在**显示层**，与账号 tier 无关 → 免费/Pro 看到相同模型。 |
| #1 未路由模型走官方 Cascade，仅被路由的走通道 | **已具备** | `dao_router.shouldRoute(uid)` 仅对路由表内 uid 返回 true；其余 `route()` 返回 false → source.js 原路透传官方。账号安全 = 零冲突 (只改路由表内模型)。 |
| #2 第二面板 = 仿 CC Switch 加渠道 | **已具备** | `extension.js` `_EA_PRESETS` 14 家 (DeepSeek/GLM/Kimi/OpenAI/Anthropic/Gemini/Grok/NewAPI…) + 预设下拉自动填 baseUrl/默认模型；Provider CRUD (增/删/编辑)。 |
| #2 第三面板 = 连线面板，右侧自动拉渠道可用模型 | **已具备** | `$eaRight` 渲染可用模型 + `eaDrawWires()` SVG 连线 + `/origin/ea/discover-models` (RPC 实证发现) + `/origin/ea/probe`。 |
| #3 默认首通道 = SWE 基础版 (仅通道检测) | **部分** | 无为模板默认只路由 `MODEL_SWE_1_6_FAST`；"纯检测通道(不落第三方)"概念需在 UI 明确标注为测试通道。 |
| **#3 首次加第三方 → 默认把 SWE 1.6 Fast 路由到该渠道 (仅一个模型) + 反馈** | **✅ 本次实现** | `dao_router.hotAddProvider`: 首个 provider 且 SWE 1.6 Fast 未路由时，自动建 `MODEL_SWE_1_6_FAST → {provider, 第一个模型}`，返回 `autoRoute`；不覆盖已存在路由。`extension.js` 收到 `autoRoute` 后在面板提示「默认路由 SWE 1.6 Fast → X」。已写 Node 单测 9/9 通过。 |
| #4 后端与官方 100% 同步: 引用历史对话 / 子代理快速上下文 / 隔离替换提示词 / 底层工具 | **待深化** | `cascade_wire.js` 已做 protobuf 编解码 + SP 隔离替换 (帛书) 已在 source.js；但「引用历史对话」「子代理快速上下文」等官方特有工具对第三方模型的透传/重建需逐个验证补齐 (依赖 `adapters.js` 工具 schema 透传 + `allowMcpTools`)。**需真实 provider key 做端到端验证。** |

### Phase 1 剩余 (proxy-pro)
- [ ] #3 测试通道在 UI 标注 (SWE 基础版 = 纯通道检测，发送消息后反馈"通道已打通")
- [ ] #4 用真实第三方 key (如 DeepSeek) E2E 验证: Cascade 发消息 → 路由到第三方 → 工具调用/引用/子代理是否完整回传；逐项补齐 `adapters.js`
