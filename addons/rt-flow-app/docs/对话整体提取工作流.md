# Devin Cloud 手机版 · 对话整体提取工作流

> 最高效地把某账号（含额度耗尽账号）某条 / 全部对话的**完整内容 + 文件**整体提取出来。

## 核心认知

**提取只读取账号已有的历史数据，完全不消耗额度。**
额度耗尽（credits = 0）的账号，照样能用「邮箱 + 密码」登录拿到 `auth1`，再整体导出全部对话与文件。额度限制的是「新建会话 / 发新消息」，不限制读取历史。

## 提取三要素

| 要素 | 取法 |
|------|------|
| 账号（email + password） | 已在号池则直接用；否则 `addAccount {email, password}` |
| `auth1` / `orgId` | `login {email}` 自动解锁（不消耗额度） |
| 会话 ID（`sessionId` / `devin_id`） | `listSessions {id}` 列出，取每条的 `devin_id` |

会话 ID 形如 `devin-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`，也即 `https://app.devin.ai/sessions/<id>` 里的 `<id>`。

## 方式 A · 最高效：一行整体提取（推荐）

```jsonc
// 单条会话：一次拿齐 元数据 + 完整对话 md + 工作日志 md + 文件清单，并落盘共享保险箱
{ "cmd": "extractConversation", "id": "<email>", "sid": "<devin_id>", "save": true }

// 要 MD+ZIP（含产出文件夹）：加 zip:true → 返回 zipB64（base64，解码即整包 ZIP）
{ "cmd": "extractConversation", "id": "<email>", "sid": "<devin_id>", "save": true, "zip": true }

// 整账号全部会话：
{ "cmd": "extractAccountConversations", "id": "<email>", "limit": 200, "includeMd": true }
```

`extractConversation` 返回字段：

| 字段 | 含义 |
|------|------|
| `conversationMd` | 完整对话（USER / DEVIN 逐条） |
| `worklogMd` | 工作日志（文件编辑 / 命令 / 浏览器 / 计算机操作全程） |
| `detail` | 会话元数据（标题 / 状态 / 时间…） |
| `files` / `fileCount` | 附件清单（name / url / path） |
| `saved` | `save:true` 时落盘的文件名（写入共享保险箱 `Documents/DevinCloud/`） |
| `zipB64` / `zipName` / `zipFileCount` | `zip:true` 时返回：整包 ZIP 的 base64（含 `对话_人类可读.md` + `工作日志.md` + `_meta.json` + `files/<产出文件>`）。**本地有产出文件即「MD + ZIP」形态**；云端 Agent base64 解码落盘即可。切号面板每条对话行的 📦 按钮等价于此一键下载到手机「下载」目录 |

## 方式 B · 分步法（需要精细控制时）

1. `login {email}` → 解锁 `auth1`（额度耗尽也可）。
2. `listSessions {id, limit:200}` → 拿全部会话与 `devin_id`。
3. 逐条 `exportSession {id, sid, kind:"conversation"}` 与 `kind:"worklog"`。
4. `sessionMessages` / `sessionDetail` 兜底取消息与附件。
5. 需落地手机：加 `save:true` 写入共享保险箱（卸载 / 重装不丢），或 `phoneWriteFile` 指定路径。

## 方式 C · 浏览器自动化提取（无 `auth1` / 只想要页面所见即所得）

若不便登录，可用「板块一 · 浏览器自动化」直接读已登录的网页：

1. `browseOpen {url:"https://app.devin.ai/sessions/<id>", account:"<email>"}` — 用该号上下文新开页，**不打扰用户当前页**。
2. `browseWaitForElement` 等内容加载 → `browseExportMd {tabIndex, save:true}` 导出整页 Markdown。
3. 或 `browseGetDom` 取完整 DOM 自行解析。

## 方式 D · 拖拽提取（页面内 · 最快：一条会话整包拖入新窗口）

理想交互：在切号面板的「对话追踪」里**长按一条会话 → 拖到任意网页标签松手**，该会话的 session md 会被整包注入目标页（上传框 / 拖放区），实现「拖一下即把整段对话 + 文件带进新窗口」。

- 底层复用 v0.15.9 的下载拖拽通道（ClipData 全局拖拽 + WebView `OnDragListener` → 注入 `input[type=file]` / `drop` 事件）。
- 程序化等价：`extractConversation {save:true}` 落盘后，在下载面板长按该 `.md` 拖入页面。

## 提取产物去向

- `save:true` → 共享保险箱 `Documents/DevinCloud/`（经 `Native.writeFile`），卸载 / 重装不丢。
- 远程取回：`phoneReadFile {path}` 读回内容，或 `phoneShare` 系统分享。
