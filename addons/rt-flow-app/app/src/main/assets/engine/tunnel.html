🛰️ P2P 直连 · WebRTC 真打洞（加法式·可选升级）

本源不动: curl/HTTP 控制端照常用上面的隧道; 这是给「愿意升级」的客户端的真 P2P 通道 —— 仅借隧道交换一次 SDP 信令, 之后 RPC 走 P2P DataChannel(公共 STUN 打洞), 不再经任何边缘中转, 延迟更低、更去中心化。复用同一 Token/命令集。
用法: 控制端打开 p2p-client.html, 填本设备上面的公网入口 URL + Token + Session 即可直连。对称 NAT 直连不通时可在客户端填自有 TURN(默认零依赖纯 STUN)。
